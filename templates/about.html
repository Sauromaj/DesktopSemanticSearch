About

Semantic Document Search

A powerful document search application that uses semantic search to find relevant documents based on meaning, not just keywords.

Version 1.0.0

How It Works

Semantic Document Search uses advanced natural language processing to understand the meaning behind your search queries and document content.

  1. Document Processing: When you upload documents, the system extracts text and splits it into manageable chunks.
  2. Embedding Generation: Each chunk is converted into a numerical vector (embedding) that represents its semantic meaning.
  3. Vector Indexing: These embeddings are stored in a FAISS vector database for efficient searching.
  4. Semantic Search: When you search, your query is converted to an embedding and compared to document embeddings to find the most semantically similar content.
  5. Results Ranking: Results are ranked by similarity score, showing the most relevant documents first.

Supported File Types

  • PDF Documents

    Extract text from PDF files including reports, papers, and forms

  • Word Documents

    Process DOCX files with formatting preserved

  • Excel Spreadsheets

    Index data from XLSX files across multiple sheets

  • CSV Files

    Process comma-separated value files with tabular data

Technology Stack

  • Frontend: Flask, Bootstrap 5, HTML/CSS/JavaScript
  • Natural Language Processing: Sentence Transformers, HuggingFace
  • Vector Database: FAISS (Facebook AI Similarity Search)
  • Document Processing: PyPDF, docx, openpyxl, pandas
